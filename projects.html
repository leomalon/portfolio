CotiLink (2025)

Visita el proyecto: cotilink.up
CotiLink es una aplicación web diseñada para que los administradores de negocios puedan dar seguimiento a sus cotizaciones. Ofrece una interfaz clara y funcional para registrar, visualizar y actualizar el estado de cada cotización, facilitando así la gestión comercial y la comunicación con los clientes.

🎯 Objetivo y Motivación

Este proyecto nació a partir de una necesidad concreta: muchos administradores de negocios o personas del área de ventas pierden oportunidades de venta por no tener una herramienta clara que les permita organizar y hacer seguimiento de sus cotizaciones. Cotilink resuelve este problema al centralizar todo en una sola plataforma accesible y fácil de usar.

🔄Flujo de Desarrollo

Planeamiento y diseño

Inicialmente, investigue sobre herramientas actuales, entreviste a proveedores con los que había trabajado y consulté por las herramientas actuales que usan para hacerle seguimiento a sus cotizaciones. La mayoría indicó que no tenía un sistema de seguimiento, otros indicaron que tenían un sistema que era muy caro y que al ser independientes no podian pagarlo mensualmente y otros proveedores indicaron que encontraron herramientas, pero eran complicadas de usar.

Luego definí mi hipótesis de valor y de crecimiento para mi idea siguiendo los conceptos del método Lean Startup.

Después, definí el MVP (Minimum Viable Product) con requisitos mínimos funcionales como el sistema de autenticación, la creación y eliminación de cotizaciones, el campo para buscar cotizaciones y el envío automático de correos electrónicos.

Finalmente, escogí las herramientas y el framework a usar para desplegar la página web.

🔄Flujo de Desarrollo

Planeamiento y diseño

Definí las secciones principales: login, página de registro, tabla de cotizaciones principal, página de cancelación, página de creación de cotizaciones y página de modificación de cotizaciones. Utilicé FIGMA para crear los layouts y el flujo de la página.

Desarrollo del Backend

En este punto inicie el desarrollo de la aplicación, es decir, escribir código. Primero definí la estructura de las carpetas según las aplicaciones definidas. El framework escogido para el desarrollo web fue Django. Para el sistema de autenticación se utilizó "django-allauth" especialmente para manejar el login con "google".

Luego modelé la estructura de la base de datos en el que definí los campos, los tipos de datos aceptados y las relaciones entre tablas. A partir de las tablas de la base de datos generé las plantillas de los forms, la lógica de los forms para crear, leer, actualizar y eliminar datos en la base de datos.

Utilicé los default views como "ListView", "CreateView", "UpdateView" y "DeleteView" que luego personalice según la lógica de mi aplicación.

Desarrollo del Frontend

Organicé y desarrolle los templates de html, junto con los archivos de estilo css. Dentro de los templates maneje variables de contexto para mostrar los datos de las bases de datos, según el usuario. Además, dentro dentro de los templates utilicé template tags para cargar los archivos estáticos y extener plantillas base.

Deployment de la aplicación

Para el deploy de la aplicación utilicé Railway. Publiqué mi código base en mi repositorio deGitHub. Luego en Railway generé una base de datos "PostgreSQL" y conecte la base de datos con mi repositorio de Github. Finalmente, configure mis variables de entorno para utilizarse en producción, como la contraseña del email host, del acceso a la base de datos, etc.

ValenDesk (2024)

Visita el proyecto: leomalon.github.io/valentina
Esta es una aplicación web creada para acompañar a mi hermana en su primer ciclo universitario. Incluye herramientas simples y funcionales como una app de tareas, una app para tomar notas, y un sistema básico de login. El objetivo fue brindarle una plataforma personal para organizar su día a día académico.

🎯 Objetivo y Motivación

Quise crear un espacio digital personalizado para ayudar a mi hermana a mantenerse enfocada y organizada desde el inicio de la universidad. En lugar de depender de múltiples aplicaciones, esta página reúne lo esencial en un solo lugar, con una interfaz limpia y fácil de usar.

🔄Flujo de Desarrollo

Planeamiento y diseño

Definí las secciones principales: login, espacio de registro de tareas y espacio de registro de notas. Luego ingresé a FIGMA para diseñar las vistas de las páginas. Me inspiré en los personajes favoritos de mi hermana y sus colores favoritos. Además, diseñe el flujo de las páginas.

Desarrollo del sistema de login

Al ser una aplicación simple sin datos sensibles, la validación de usuario fue básica usando JavaScript y almacenamiento local. No se usó una base de datos o un sistema de encriptado para almacenamiento de contraseñas.

Desarrollo de la app de tareas

En esta página se permite agregar tareas, nuevas, marcas tareas completadas o pendientes. También permite filtrar tareas por estado, eliminar tareas o todas la vez.

En esta página el historial de tareas se guarda en el "Local Storage", lo que es útil en este caso, ya que al no funcionar con una base de datos, las tareas siguen ahí incluso si el usuario cierra sesión o cierra el navegador.

Al cargar la página se llama la función "showTodos()" para mostrar las tareas en pantalla. La función recorre todas las tareas almacenadas en "todosJson". Las tareas se muestran como elementos HTML con un "checkbox" para marcar el estado, el nombre de la tarea y un botón para eliminar.

Para agregar una nueva tarea se tiene la función "addTodo()". Cuando el usuario escribe una tarea y presionar "Enter", se limpia el campo de entrada, agregar la nueva tarea al inicio de la lista con estado "pending" y finalmente, se actualizar el "localStorage" y se vuelve a mostrar la lista.


Para actualizar el estado de una tarea se usa la función "updateStatus()". Al hacer click en el "checkbox", la tarea se marca y se cambia su estado a "completado" y si se desmarca, se vuelve a "pending" y se quita la clase. Finalmente, se actualiza el "LocalStorage".

Para borrar una tarea se usa la función "remove()" y al hacer click se elimina la tarea, según su índice en el arreglo "todosJson", se actualiza el "localstorage" y se vuelve a renderizar.

Para filtrar las tareas y mostrarlar se usa la función "filters". Al hacer un click en un filtro, se activa el filtro y muestra las tareas que coinciden con el filtro seleccionado y se vuelve a llamar "showTodos()".

Finalmente, para eliminar todas las tareas se tiene la función "deleteAllButton". Se limpia el arreglo "todosJson", se limpia el "localStorage" y se actualiza la vista para mostrar la lista vacía.

Desarrollo de la app de notas

En esta página se permite agregar notas con título y una pequeña descripción, como si tuvieras un board con sticky notes.

Koi timer (2025)

Descarga el proyecto: koi-timer
Esta es una aplicación de escritorio creada para poder controlar mis tiempos de concentración. Incluye una de mis piezas favoritas en piano de Franz Liszt como música de fondo y una linda animación de peces koi. El sistema de este timer está basado en la teoría de pomodoro de bloques de concentración seguidos de bloques de descanso.

🎯 Objetivo y Motivación

Quise crear un temporizador funcional capaz de crear un espacio estimulante para concentración y estudio. La idea surgió a partir del temporizador que estaba usando como parte de las aplicaciones de Microsoft que si bien es funcional, estéticamente no era de mi gusto, me parecía plana y aburrida, por lo que decidí crear una por mi cuenta.

🔄Flujo de Desarrollo

Planeamiento y diseño

Definí la lógica de funcionamiento del temporizador tomando en cuenta la teoría de bloques de tiempo de concentración. Luego ingresé a FIGMA para diseñar todas las vistas del temporizador, el aspecto del timer, fuentes y colores.

Desarrollo de la lógica y backend

Para este proyecto tuve que instalar node.js para poder correr javascript fuera del navegador. Además, tuve que instalar el framework electron.js. En este caso utilice la asistencia del editor de código Cursor AI para agilizar mis tiempos de programación. La lógica del timer funciona de la siguiente manera: Si el tiempo seleccionado es menor de 25 minutos, solo existe un bloque único de 25 minutos sin descanso. Si el tiempo es igual a 25 minutos, existe un bloque de tiempo de 25 minutos más 5 minutos de descanso. Si el bloque es mayor a 25 minutos entonces el tiempo total se divide en bloques de 25 minutos con 5 minutos de descanso entre bloques, en caso el tiempo total no sea múltiplo de 25 minutos, se divide el tiempo total entre 25 minutos y el número entero será el número de bloques y el residuo será un bloque completo al final.

Para configurar la aplicación generé un archivo package.json en dónde liste los scripts del proyecto, nombre de la aplicación y las configuraciones para generar el ejecutable.

Desarrollo del estilo

Para este proyecto utilice mis habilidades de html, javascript y CSS para programar el frontend de la aplicación, ya que electron.js empaqueta un navegador y node.js, lo cual hace más fácil definir el layout y el diseño de las vistas.

Generación de ejecutable e instalador

Utilicé electron.js para empaquetar la aplicación en un ejecutable y para generar el instalador usé Inno Setup Compiler.

Ashventure (2024)

Descarga el proyecto: asventure
Este es un juego que hice como regalo inspirado en Fire boy and Water girl. El personaje principal tiene que conseguir las monedas necesarias en las diferentes plataformas y deberá tener cuidado con el fuego que está por todos lados. Al obtener todas las monedas el juego termina muestra una serie de fotos del recuerdo.

🎯 Objetivo y Motivación

Quise crear un regalo diferente a los que se suelen dar, entonces decidí combinar mi pasión que es programar y algo que me gusta que son los videojuegos.

🔄Flujo de Desarrollo

Planeamiento y diseño

Definí la lógica del juego, niveles, dificultad y modo de jugar. Para el diseño busqué imágenes de internet para el fondo y las plataformas. Para el diseño del personaje principal tome una de mis fotos favoritas y la pasé a CHATGPT para que generara un avatar pixeleado para darle ese aspecto de juego.

Desarrollo del juego

Para este proyecto utilicé python y la librería pygame para facilitar el desarrollo del juego. Para el diseño del juego utilicé FIGMA y además usé CHATGPT para realizar las imágenes en formato pixeleado. Finalmente, empaqueté el juego en un ejecutable usando la librería auto-py-to-exe.
